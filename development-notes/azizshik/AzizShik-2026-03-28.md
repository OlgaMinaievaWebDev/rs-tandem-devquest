## Что было сделано

- Реализовал систему toast-уведомлений для замены всех нативных `alert()`.
- Создал новый компонент `toast.ts` с функцией `showToast(message, type)`.
- Добавил поддержку двух вариантов: `error` и `success`.
- Реализовал плавную анимацию появления/исчезновения с использованием
  `requestAnimationFrame`.
- Подключил стили для toast-контейнера и разных типов уведомлений
  (позиционирование bottom-right).
- Полностью заменил все вызовы `alert()` в `main.ts` на `showToast()`.
- Добавил удобные хелперы `showError()` и `showSuccess()`.
- Обеспечил авто-закрытие уведомлений через 3 секунды и возможность закрытия по
  клику.

## Решения

- Использовал `requestAnimationFrame` для корректного запуска entrance-анимации
  после добавления элемента в DOM.
- Сделал toast-систему лёгкой, без внешних библиотек и с минимальным
  воздействием на производительность.
- Выбрал позиционирование `bottom-right`, которое хорошо сочетается с общим
  стилем приложения.
- Сохранил единый стиль кода и добавил типизацию для `ToastType`.
