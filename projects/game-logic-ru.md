# 🎮 DevQuest — Логика Игры

## 1. Общая концепция

DevQuest — это симулятор рабочего дня разработчика.

Игрок:

- Регистрируется и создаёт профиль
- Выбирает специализацию (Skill)
- Проходит игровые дни
- Выполняет мини-задачи
- Управляет показателями **Stress** и **Authority**
- Может обращаться за помощью к Senior Developer (AI)

---

## 2. Игровой цикл

Каждый игровой день:

- Доступны **3 мини-игры**
- Нужно пройти **2 из 3**, чтобы завершить день
- На каждую задачу действует **отдельный таймер**
- Сложность увеличивается с каждым днём

Если выполнены 2 задачи:


TASK_FINISHED
→ statsManager.apply()
→ dayManager.increment()
→ emit DAY_COMPLETED


После `DAY_COMPLETED` игрок переходит к следующему дню.

---

## 3. Система навыков (Skill System)

Игрок выбирает специализацию (например, Frontend, DevOps и т.д.).

Навык влияет на:

- Тип задач
- Формулировки заданий
- Реплики Босса
- Контекст AI-промпта

Навык меняет контент, но не основную механику.

---

## 4. Система показателей

### 🔴 Stress

- Увеличивается при ошибке
- Увеличивается при истечении таймера
- 100% → burnout → конец игры

### 🟢 Authority

- Увеличивается при правильных ответах
- Может уменьшаться при злоупотреблении подсказками
- Используется для ощущения прогрессии

---

## 5. Типы задач (Widgets)

Поддерживаются разные типы мини-игр:

### 5.1 quiz

- Вопрос
- 4 варианта ответа
- Без текстового ввода

### 5.2 code_review

- Текстовый чат
- Игрок вводит ответ вручную

### 5.3 drag_and_drop

- Интерактивная зона
- Расстановка элементов в правильном порядке

Все виджеты возвращают единый формат результата:

```text
{
  outcome: "correct" | "wrong" | "timeout",
  userAnswer: string
}
```

Это обеспечивает единообразную обработку результатов.

### 6. Senior Developer (AI Help)

Игрок может задать вопрос Senior Developer.

Ограничения:

Лимит запросов в день
или
Cooldown между сообщениями
# AI используется для:
Генерации задач
Генерации подсказок
Имитации общения

### 7. Прогрессия

С каждым днём:
Увеличивается сложность
Задачи становятся глубже
Ошибки наказываются сильнее
Реплики Босса становятся строже
Игра создаёт ощущение дедлайна и профессионального роста.

### 8. Архитектурный принцип логики

Принцип разделения ответственности:
UI только отображает
Game Logic управляет правилами
Store хранит состояние
EventBus передаёт события

Пример потока:

Игрок отправил ответ
→ TASK_FINISHED
→ statsManager обновил Stress / Authority
→ dayManager проверил правило 2 из 3
→ emit DAY_COMPLETED (если условие выполнено)

Это обеспечивает модульность и масштабируемость.
