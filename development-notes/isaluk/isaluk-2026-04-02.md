# Дата: 2026-04-02

- **Что было сделано:** Настроила безопасный AI-прокси через Supabase Edge Functions (Issue #10). Теперь наш секретный API-ключ Gemini надежно спрятан на бэкенде, а не светится в браузере.
  1. Создала функцию на Deno, настроила CORS-заголовки для работы с фронтендом.
  2. Загрузила секреты (`GEMINI_API_KEY`) в облако Supabase.
  3. Полностью отрефакторила `AIService`: теперь он умеет запрашивать разные типы задач (квизы, чаты), используя один приватный метод для связи с Edge Function.

- **Проблемы:** 1. _Цепочка ошибок:_ Прошла через 401 (Unauthorized) от Supabase и 400 (Bad Request) от Google из-за устаревшего названия модели в запросе. 2. _Грязный текст:_ Gemini возвращал спецсимволы вроде `&lt;`, которые ломали верстку. 3. _Безопасность (Security Debt):_ Чтобы функция заработала быстро, пришлось деплоить её с флагом `--no-verify-jwt`. Это значит, что сейчас любой, кто узнает URL функции, может её вызвать и тратить наши токены.

- **Решения:** 1. _Дебаг:_ Передеплоила функцию с актуальной моделью `gemini-2.5-flash` и временно отключила проверку JWT для отладки. 2. _HTML-хак:_ Написала утилиту `decodeHTML`, которая использует скрытый элемент `<textarea>` для мгновенного превращения спецсимволов в нормальный текст. 3. _План по защите:_ Сформулировала задачу на будущее — убрать флаг `--no-verify-jwt` и настроить передачу `access_token` пользователя из `watchAuth`, чтобы API был доступен только авторизованным игрокам.

- **Мысли / Планы:** Сегодня был мощный full-stack день. Настроить связь Фронтенд -> Edge Function -> Gemini API. Бэкенд-часть готова, команда разблокирована. Дальше хочу поработать над системой баллов и оживить наш сайдбар
